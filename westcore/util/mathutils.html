inline double clamp(double value, double min, double max)

Utility method for clamping a value between a min in max.

In the PID controller, an ever-occurring problem was the over-saturation of either the linear or angular controller, causing the robot to either spin rapidly (angular is too high) or move straight without any angular correction (linear is too high). USing this method, the velocity of the linear can be clamped below 12000 in order to guarantee angular can correct any potential heading error.

Parameters

  • value – The value to be clamped

  • min – The minimum of the clamp. If the value is below the min, min is returned.

  • max – The maximum of the clamp. If the value is above the max, max is returned.

Returns

A clamped value between min and max

inline double normalizeAngle(double deltaAngle)

Normalizes a delta angle to prevent leaps in value between 0 and 360.

For example, if the robot was currently at 340 degrees and wanted to turn to 0 degrees, this will calculate a difference of 20. More importantly, when the heading shifts from 360 to 0 there will be no sudden drastic change of the heading, thanks to atan2 being used.

This was a problem with the original version of this program, in which the robot might overshoot and suddenly move in sporadic directions.

Parameters

deltaAngle – The size of the angle between two degree measurements, or target - current

Returns

The normalized angle of deltaAngle
